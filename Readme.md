# Steam 游戏数据分析系统

从 Steam 数据采集、存储到分析展示的全链路 Java 工具。支持按月批量爬取新游数据、多维度分析查询，并提供完整的控制台交互菜单。

**项目成果**：从 2026 年 6 月 128 款 Steam 新游中筛选出 5 款真正有创新内核的精品游戏，并制作了数据解读视频。

## 快速运行

### 环境要求
- Java 8+
- MySQL 8.0+

### 文件说明
SteamSQL/
├── src/                         # 源代码
│   ├── control/                 # 控制台菜单控制层
│   ├── dao/                     # 数据访问层
│   ├── entity/                  # 实体类
│   ├── service/                 # 业务逻辑层
│   ├── util/                    # 工具类
│   ├── steamApi.properties      # 配置文件（需修改）
│   ├── sql/                     # 数据库初始化脚本
│   ├—— translate_enum           # 与英文评价对应的翻译(如OVERWHELMINGLY_POSITIVE=好评如潮)         
├── lib/                         # 依赖 JAR 包
├── test/
│   ├── SteamSQL.jar             # 可执行 JAR
│   └── run.bat                  # Windows 启动脚本
└── README.md                    # 项目说明

主菜单
├── 1. 爬取游戏商店数据
│   ├── 1. 爬取某一月份的新发布游戏数据（批量）
│   └── 2. 爬取单个游戏的游戏发布数据（单条）
├── 2. 查看游戏数据
│   ├── 1. 根据月份查询（分页 + 详情）
│   ├── 2. 查询免费游戏（列表 + 详情）
│   └── 3. 查询好评率 Top10（列表 + 详情）
├── 3. 修复日期字段
├── 4. 初始化数据库（当前状态：已初始化/未初始化）
├── 5. 翻译评论描述字段
└── 0. 退出

⚠️ 注意事项
请确保 MySQL 服务已启动且连接信息正确。

首次使用必须先执行 4. 初始化数据库。

爬取数据时请合理控制频率（代码已内置 300ms 延时），避免触发 Steam 限流。

可点击test/run.bat用Windows命令行终端使用，也可导入JAVA项目后调用Fulltest.run()使用

配置文件中的 checkUrl 字段用于检测数据库状态，无需修改。

如果使用 JAR 包运行，配置文件可放在 JAR 同级目录或 src/ 目录下，程序会自动识别。

run.bat 默认在 test/ 目录下运行，如果修改了目录结构，请相应调整配置文件路径。
